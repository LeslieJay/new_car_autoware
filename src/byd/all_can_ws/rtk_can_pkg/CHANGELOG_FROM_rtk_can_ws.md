# RTK 包改动说明（口语版）

旧目录：`rtk_can_ws`  
新目录：`rtk_can_pkg`（现在真正该用的）

一句话说清楚：  
**以前定位和整车控制搅在一起；现在单独做定位+差分，更干净、更能跑通。**

---

## 新旧到底差在哪

| | 旧包 | 新包 |
|--|------|------|
| 干什么 | 定位里还夹着车控、电池、发送队列 | 只做定位、姿态、IMU，外加六分差分 |
| 用哪个口 | can0/can1 设计乱，还经常搞混 | 固定用 can1 |
| 差分 | 没有 | 有：车上定位上报，差分数据再灌回设备 |
| 还发不发车控话题 | 建了但基本没真正发出去 | 不发了，车控交给 `can_ws` |

---

## 修了哪些毛病

1. **CAN 口名字搞反了**  
   叫 can0 的函数，实际拿的是 can1。新包不会再这样。

2. **配置可能根本加载不上**  
   配置文件里的名字和程序节点名对不上。新包名字统一了。

3. **很多代码编译了但没用**  
   比如发送队列、电池模块编进包，启动时根本没接上。新包直接去掉。

4. **数据不完整也照样往外发**  
   旧包长度检查被关掉了。新包先检查再发布。

5. **时间戳不准**  
   旧包常用电脑当前时间；新包优先用设备帧里的定位时间。

6. **IMU 轴向直出，没按协议纠正**  
   新包按协议做了轴和正负修正。

7. **一堆备份文件、调试脚本、重复定义**  
   比如带 copy 的头文件、试跑脚本，新包都不要了。

---

## 删了哪些东西

从旧包里拿掉、新包不再保留的：

- 车端发送队列相关代码
- 电池充电相关代码
- 挂车参数、旧 AGV 配置
- `can_frame` 的拷贝文件
- 调试用的 py 脚本、编辑器配置
- 航向分析脚本（没迁过来）

一句话：**凡是跟“定位差分”无关、或者编了却没接线的，都清掉。**

---

## 新包多了什么

- 六分差分能力（证书、库、回灌差分数据）
- 定位速度相关输出（以前没有）
- 原始 CAN 日志（可留档、超了会裁）
- 关程序时更规矩：先停接收，再收尾关口

另外，报文 ID 也换了新协议：  
位置/姿态用新的，差分回灌也有单独通道。旧包那套 ID 不能直接拿来用。

---

## 换到新包时要注意

1. 用新包启动，不要再跑旧的那套  
2. 设备侧报文要按新协议来  
3. 差分库文件和证书要带上  
4. can1 要先配好再启动（程序不负责帮你起网口）

---

## 还没做完的

- 差分账号密码还写死在程序里，以后最好改成配置
- 有些文件还重复拷了两份
- 网口参数里还有没用上的项

---

## 最后一句

**平时联调、上车，用 `rtk_can_pkg`。**  
`rtk_can_ws` 只当旧东西对照，别再当正式包用。
